# git提交标签

默认在前缀使用`<>`框出标签，用来便捷区分提交内容的类型，支持的标签如下所示

## 核心标签

| 标签 | 描述 | 使用场景 |
|------|------|----------|
| **<feat>** | 新功能 | 添加新功能、新模块 |
| **<fix>** | 修复bug | 修复代码缺陷、逻辑错误 |
| **<chore>** | 维护任务 | 配置文件、构建脚本、依赖更新 |

## 其他标签

| 标签 | 描述 | 使用场景 |
|------|------|----------|
| **<docs>** | 文档更新 | 修改README、API文档、注释 |
| **<test>** | 测试相关 | 添加/修改测试用例、测试配置 |
| **<refactor>** | 代码重构 | 优化代码结构，不改变功能 |
| **<style>** | 代码样式 | 格式化、空格、分号等样式调整 |
| **<perf>** | 性能优化 | 提升性能的代码更改 |
| **<build>** | 构建系统 | Dockerfile、CI/CD配置、打包脚本 |
| **<ci>** | 持续集成 | GitHub Actions、Jenkins配置 |
| **<deps>** | 依赖更新 | 添加、更新、删除项目依赖 |
| **<revert>** | 撤销更改 | 回滚之前的提交 |
| **<clean>** | 清理代码 | 删除无用文件、代码、注释 |
| **<db>** | 数据库更改 | 迁移脚本、表结构变更 |
| **<wip>** | 进行中工作 | 临时提交，功能未完成 |
| **<init>** | 初始化项目 | 项目首次提交、新模块初始化 |
